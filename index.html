MadEvolve 复现
LLM-Driven Evolutionary Discovery · 论文分析与代码复现
思想起源

MadEvolve 思想来源于 DeepMind 的 Alpha-Evolve。核心流程：LLM 写程序 → 自动化评估代码 → 好程序放到程序库 → LLM 基于程序库的程序迭代生成下一代程序。类似于遗传算法，核心在于环境（考核标准）不变。

突变来自于 LLM，适应度函数是回测结果（Fitness Signal: 去掉手续费和冲击成本的 PnL），程序库对应种群。进化目标分两个：set_target（输出归一化目标仓位数值 [-1,1]）即使用因子；set_limit_order（输出具体限价单指令）即执行策略；default_calcset 是生成因子即特征工程。

岛屿模型 & Map-Elites

使用岛屿模型：5 个种群独立进化，每 5 代进行 10% 的精英迁移。工程实现上加了全局精英策略档案来保存固定容量的全体最优策略。

Map-Elites 模型强制保持多样性，每个格子保留格子内最强的策略（代码复杂度和多样性）。

LLM 使用多模型路由：小模型做增量优化，大模型做结构创新。生成模式有差分补丁（~70%）、完整重写（~30%）、多父代合成。

数据与策略

标的：BTCUSD 一分钟 OHLCV 柱线，数据来源 Polygon.io。一分钟级别 BTCUSD 市场近乎有效，价格变动近乎完全取决于新信息，走势近似随机游走，OHLCV 信息最多只能解释 0.2%-0.4% 的价格变动。

策略类型：被动限价单策略（每分钟一次决策）。被动挂单手续费低，可吃流动性补贴；低频省手续费，网络要求低且大概率没滑点。

数据分割：2022-2023 训练 alpha 模型，2024 验证，2025.1.1-10.10 测试。训练集训练策略学规律，验证集调超参提供无偏调整，测试集展示泛化能力。

实验设计与结论

5 轮实验：run1（目标仓位逻辑）、run2（下单逻辑）、run3（1+2 联合）、run4（特征工程，适应度改为预测质量复合分数 = 0.4×R² + 0.3×IC + 0.3×ICIR/5）、run5（完整流程：生成因子→使用因子→执行策略）。

核心结论：① 执行层的优化贡献最大；② run5 策略和特征联合优化的收益质量很高；③ 好特征要配好策略；④ run5 胜率从 68.6% 降到 49.8% 但 Sharpe 仍很高——模式变成了"少赚大钱"。

关键技术细节
  • EVOLVE-BLOCK 标志划定可变异区域，防止 gaming the scoreboard
  • Adam 参数优化被移除：在量化中会放大多重检验问题（p-hacking），偏向过参数化结构
  • 选择 PnL 而非 Sharpe 作为适应度：Sharpe 可通过选择性交易刷高，冲击调整 PnL 内含成本惩罚
  • 过拟合检验：PnL 保留率、Sharpe 保留率、胜率变化 + Z-score 随机最优对比
  • run4 因子质量评估：用固定 α=0.5 的 Ridge 回归测试 1/10/100/1000 分钟收益预测能力
  • 日度 IC/ICIR：避免高频信号被手续费吃光，样本间更接近独立
与 Claude Code 的比较
  • CC 没有额外结构，MadEvolve 有种群多样性机制（岛屿模型 + Map-Elites）
  • CC 有想法树，搜索更系统完整；CC 生成的因子质量很高，R² 约 5 倍提升
  • CC 限制因子 10 个（贪心选择 + 低相关性），MadEvolve 有 77 个（Ridge 回归分散权重）
  • CC 表现高度依赖提示词，论文表示无法明确判断哪个框架更好
